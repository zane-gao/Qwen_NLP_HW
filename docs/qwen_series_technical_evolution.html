1. 总览：Qwen 的四条技术主线

Qwen 的发展不是简单堆参数，而是通用 LLM、效率架构、显式推理、多模态原生化四条路线互相反哺。

通用 LLM 能力扩展从 Qwen-7B/14B/72B 到 Qwen3.6，持续提升中英/多语、指令跟随、长文本、代码和数学。
效率架构扩展从 dense Transformer 到 GQA、MoE、Hybrid Attention、Gated Delta Networks，目标是更低训练和推理成本。
推理与 Agent 显式化从工具调用、代码解释器到 QwQ/QVQ，再到 Qwen3 hybrid thinking 和 Qwen3.6 thinking preservation。
多模态原生化从 Qwen-VL/Qwen-Audio 到 Qwen3-VL、Qwen3-Omni、Qwen-Image、Qwen-VLA，统一文本、图像、音频、视频和动作。

2. 主线逐代分析

2.1 Qwen 初代：开放中文/中英大模型的工程化起点

重点解决的问题：中文与英文基础能力、开源模型工具使用、训练/部署/微调工程链路。

专门技术：最多 3T token 多语多领域预训练；1.8B/7B/14B/72B 多尺度；Base/Chat 双线；SFT/RLHF；工具调用、agent、代码解释器。

架构：decoder-only Transformer，使用 RoPE、SwiGLU、RMSNorm 等主流高效组件。

算法：高质量中英、代码、数学数据扩展；Chat 对齐；32K 长上下文逐步加入。

工程：Int4/Int8、GPTQ、KV cache 量化、LoRA/Q-LoRA、vLLM/FastChat、WebUI/CLI、Ascend/Hygon 支持。

2.2 Qwen1.5：稳定多尺寸、生态兼容与首个 MoE

重点解决的问题：把第一代强模型做成稳定模型族，覆盖更多尺寸、更统一上下文和更标准的生态接口。

专门技术：0.5B 到 110B dense 模型；统一 32K；Qwen1.5-MoE-A2.7B 首次试水稀疏专家。

架构：延续 decoder-only Transformer；MoE 将 FFN 部分稀疏化，探索总参数大、激活参数小。

算法：延续 base + chat 后训练；MoE 路由让专家在不同 token/任务上分工。

工程：Transformers 原生集成、AWQ/GPTQ/GGUF、vLLM/SGLang、llama.cpp、MLX 生态成熟。

2.3 Qwen2：规模、GQA、MoE 与 128K 长上下文

重点解决的问题：更强、更长、更省。把预训练扩到 7T token，并降低长上下文推理的 KV cache 压力。

专门技术：0.5B/1.5B/7B/72B dense 与 57B-A14B MoE；29+ 语言；最高 128K 上下文。

架构：GQA 降低 KV head 和显存带宽；MoE 在能力与激活成本之间折中。

算法：7T 预训练提升知识、代码、数学和多语；长上下文训练成为主线能力。

工程：量化、吞吐 benchmark、长上下文 serving 和多框架部署成为模型发布的一部分。

2.4 Qwen2.5：通用能力成熟与专用分支矩阵

重点解决的问题：通用模型足够强，同时 Coder、Math、VL、Omni 等专用能力形成体系。

专门技术：0.5B/1.5B/3B/7B/14B/32B/72B；18T 预训练；结构化输出、长文本、代码、数学增强；128K 与 1M 长上下文分支。

架构：沿用 Qwen2 高效 decoder 与 GQA 路线，多尺寸配置更系统。

算法：更多 STEM、代码、数学、结构化和长文档数据；Coder/Math 分支反哺 Qwen3 数据生成。

工程：成为大量 RAG、agent、代码助手基础模型；AWQ/GPTQ/GGUF、MLX、vLLM/SGLang 支持成熟。

2.5 QwQ / QVQ：显式长思考与视觉推理

重点解决的问题：数学、代码、逻辑和视觉任务需要多步推理，普通 Instruct 不能稳定展开过程。

专门技术：QwQ-Preview 展示长 CoT；QwQ-32B 通过 RL 扩展推理；QVQ-72B-Preview 探索视觉输入下的多步推理。

算法：long CoT 冷启动、规则奖励和任务奖励；为 Qwen3 thinking mode、thinking budget 和四阶段后训练提供经验。

工程：需要更长输出、更高 max token、更清晰的 reasoning content/final answer 分离。

2.6 Qwen3：thinking/non-thinking 统一，Dense + MoE 主线化

重点解决的问题：同一个模型能否同时做深度推理和快速响应，并可按任务控制 thinking budget。

专门技术：0.6B 到 32B dense；30B-A3B、235B-A22B MoE；thinking/non-thinking 模式；119 语言。

架构：Dense 与 MoE 共同主线化，用少量激活专家降低推理成本。

算法：约 36T token；三阶段预训练；四阶段后训练：long CoT cold start、reasoning RL、thinking mode fusion、general RL。

工程：SGLang/vLLM reasoning parser、Transformers chat template、Ollama/LMStudio/MLX/llama.cpp、本地和云端部署。

2.7 Qwen3-2507 / Qwen3-Next：长上下文和效率架构再分化

重点解决的问题：首发 Qwen3 的模式切换复杂、长上下文更长、深推理成本高。

专门技术：Qwen3-2507 把 Instruct 与 Thinking 分离；支持 256K 并可扩展到 1M；Qwen3-Next-80B-A3B 使用 Hybrid Attention、Gated DeltaNet、超稀疏 MoE。

架构：Hybrid Attention 处理长程依赖，Gated DeltaNet 提升长序列吞吐，MoE 降低激活成本。

工程：更依赖 serving 框架 kernel、KV cache、MoE routing、长上下文调优。

2.8 Qwen3.5 / Qwen3.6：原生多模态 Agent 与真实开发体验

重点解决的问题：把模型从聊天能力推进到真实可用的 native multimodal agent，尤其是开发、前端、仓库级任务稳定性。

专门技术：Qwen3.5-397B-A17B、122B-A10B、35B-A3B、27B、9B、4B、2B、0.8B；201 语言；早融合多模态；百万级 agent 环境 RL；Qwen3.6 thinking preservation。

架构：Gated Delta Networks + sparse MoE；统一视觉语言 foundation；35B-A3B MoE 与 27B dense 并行。

工程：Qwen Studio、Qwen API、Qwen Code、Qwen-Agent、SGLang/vLLM、llama.cpp、MLX/MLX-VLM。

3. 分支专题：从专用能力回流主线

CoderQwen2.5-Coder 通过代码继续预训练和 FIM 支持 IDE 补全；Qwen3-Coder/Coder-Next 转向 agentic coding、浏览器/终端任务和仓库级理解。
MathQwen2.5-Math 从英文 CoT 扩展到中英 CoT + TIR，并发布 Math-RM-72B，对候选解答排序。
VLQwen-VL 到 Qwen3-VL，逐步强化 OCR、文档解析、视频、2D/3D grounding、GUI control、visual coding 和 thinking 版本。
Audio / ASR / TTSQwen-Audio、Qwen2-Audio、Qwen3-ASR、Qwen3-TTS 让 Qwen 具备语音识别、音频理解、语音聊天和可控 TTS。
OmniQwen2.5-Omni 与 Qwen3-Omni 用 Thinker-Talker 连接文本、图像、音频、视频输入，以及文本/自然语音流式输出。
Image / VLA / EmbeddingQwen-Image 解决中文文字和复杂排版生成；Qwen-VLA 输出机器人动作；Qwen3-Embedding/Reranker 进入检索基础设施。

4. 技术归纳与结论

版本/分支核心问题核心技术
Qwen 初代开放中英 LLM 可用性与工程生态3T 预训练、Chat 对齐、工具调用、量化/微调/部署
Qwen1.5多尺寸稳定开放与生态兼容0.5B-110B、32K、HF 原生、首个 MoE
Qwen2更长、更省、更强多语7T、GQA、128K、57B-A14B MoE
Qwen2.5通用能力成熟与专用分支体系18T、结构化输出、Coder/Math/VL/Omni

Qwen3-Next长上下文推理成本Hybrid Attention、Gated DeltaNet、超稀疏 MoE
Qwen3.5/3.6原生多模态 agent 与真实开发体验早融合多模态、201 语言、百万 agent RL、thinking preservation

最终判断

Qwen 的长期路线是：先把开放中英 LLM 做强，再把多尺寸和工程生态做稳，然后用 Coder/Math/VL/Audio 等分支补齐专门能力，最后把推理、agent 和多模态统一回主线模型。Qwen3.5/3.6 显示出新的中心：基础模型不再只是聊天模型，而是面向真实开发、检索、视觉操作、语音视频交互和机器人动作的原生 agent foundation。
